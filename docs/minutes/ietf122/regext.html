# regext

## Summary

The regext working group meeting covered several key areas, including updates on existing documents, new work presentations on efficient ARDAP referrals and secure authcodes (OSCOSEC), and a discussion regarding a proposed charter change. The group discussed challenges related to ARDAP performance and domain transfer security, and sought feedback on potential solutions.

## Key Discussion Points

*   **RFC Status Updates:** Discussion of RFC editor queue and documents in IESG process, including the EPP mapping for DNS time-to-live values.

*   **Versioning in ARDAP:** Updates on the 03 draft, including addressing the use of bear extension identifiers and considering semantic versioning.
*   **ARE Exchanges with Extensions Media Type:** Discussion on Pavel's measurements regarding the interpretation of parameters on the current media type.
*   **ARDAP Extensions:** Discussion on simplifying the draft, specifically regarding bear identifiers. The group seeks input on simplifying the document.
*   **JS Contact in Ardap:** The document was updated to avoid better extension identifiers.
*   **Other exchanges for AR PGI registration data:** Draft presented to Cedar Ops Working Group and was well received.
*   **EPP over ACTPS and EPP over Quick:** Updates including working group document status, Gavin Brown and Zade Albana as document shepherds, and a request for reviewers.
*   **Efficient ARDAP Referrals (Gavin Brown):** Presentation on a proposal using HEAD requests to improve ARDAP referral efficiency by reducing wasted bandwidth and CPU, particularly in scenarios with low cache hit rates. Strong support for the work was expressed.
*   **OSCOSEC (Victor Kosek):** Presentation on securing domain transfers using cryptography with OSCOSEC in a backward-compatible manner. Focusing on registrant signing. The group discussed public key distribution and potential issues with DNSSEC.
*   **Charter Change Proposal:** Discussion of proposed charter changes to explicitly include protocol extensions for transporting EPP and ARDAP, aiming to clarify the working group's scope and address the eligibility of work such as EPP over HTTP and EPP over Quick.
