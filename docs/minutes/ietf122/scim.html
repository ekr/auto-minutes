# scim

## Summary

The SCIM working group meeting covered several key topics, including updates on cursor pagination, the use cases draft, and the OpenID Foundation's IPSI working group and its potential impact on SCIM.  There was a call for reviewers for the use cases draft.  Octa has implemented and found issues with the Roles and Entitlements draft, and is willing to help update it. The IPSI working group is aiming to create profiles of existing standards to improve interoperability and security, with a focus on enterprise use cases.  The SCIM working group discussed how the IPSI working group should focus on profiling rather than creating new extensions, and ways to collaborate.

## Key Discussion Points

*   **Cursor Pagination:**  The esteemed ID is moving forward with cursor pagination and incorporating feedback.
*   **Use Cases Draft:** Paolo Correira provided an update on the SCIM use cases draft.  The goal is to establish common terminology and illustrate generic and specific use cases.  More reviews are needed.
*   **Roles and Entitlements Draft:** Aaron Parecki provided an update on the Roles and Entitlements draft, noting Octa's implementation experience and identified needed tweaks. Octa is willing to update the draft.
*   **OpenID Foundation IPSI Working Group:** Aaron Parecki discussed the IPSI working group's goals to improve interoperability and security in enterprise identity.  The group aims to profile existing specifications rather than creating new ones. There's overlap with SCIM, particularly in identity lifecycle management. Collaboration between the SCIM working group and IPSI is essential. The target is to have a draft done before September 2025 for interoperability demonstrations at the Gartner IAM Summit.
*   **Enterprise Definition:**  The definition of enterprise was clarified, with the main point being control of overall application usage from a single control point.
