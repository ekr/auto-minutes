Session Date/Time: 26 Jul 2024 20:00

spring

Summary

The spring working group meeting covered document updates, rechartering, and several draft presentations. Key topics included security considerations in segment routing, validity criteria for SR policies, inter-layer network programming, micro-tap segments, SRv6 SPAN, IPv6-only networks, BFD encapsulation for SRv6 policies, aggregate header limits, and a new SRv6-based data plane for deterministic networking. The meeting emphasized the need for increased working group participation in document adoption and milestone management.

Key Discussion Points

  • Rechartering: The charter is being updated to reflect current work and the creation of the SRV6 ops working group. A key change is moving away from a fixed list of work items in the charter to managing milestones based on working group input and AD consultation. Concerns were raised about the removal of explicit mentions of security and untrusted domains.
  • Security Considerations Draft: Discussed expanding the scope description and reworking the threat model section. Open questions remain about including micro-tap segment topics.
  • SR Policy Validity Draft: Defined new validity parameters for SR policies based on comments from the community, aiming to call for WG adoption.
  • SRv6 Inter-Layer Network Programming: Presented use cases for SRv6 in integrating IP, optical, and metro transport networks. The draft proposes new SRv6 endpoint behaviors. Concerns were raised about the limited applicability, trust between networks, and if there's really a need for new behaviors vs. just leveraging existing ones.
  • Micro-Tap Segment Draft: Updates included SRv6 optimizations and support for tapping by the monitor node itself. Concerns were raised about ECMP impact, security considerations, and negative connotations of using the term "tap".
  • SRv6 SPAN Draft: Discussed SRv6 SPAN for mirroring technology, seeking compatibility with ERSPAN.
  • IPv6-only Network Draft: The group looked at segment for IPV6 over IPV6-only networks.
  • BFD Encapsulation Draft: Presented updates removing BFD echo packet encapsulation and renaming modes.
  • Aggregate Header Limit Draft: Addressed the problem of increasing header sizes in SRv6 and its impact on devices with processing limits.
  • SRv6 for Deterministic Networking Draft: Introduced a new SRv6-based data plane for deterministic networking and pre-off function.

Decisions and Action Items

  • IPv6-only Network Draft: Authors to provide references for what they're working with and show how it's different from other available options.
