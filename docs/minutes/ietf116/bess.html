Session Date/Time: 29 Mar 2023 00:30

bess

Summary

This BESS working group meeting covered a variety of topics, including document status updates, presentations of new and existing drafts, and requests for working group adoption. A major theme was the need to address existing issues in previously published RFCs and drafts, with several presentations focused on clarifications and extensions to existing procedures. The chairs emphasized the need to reduce the document backlog and encouraged more frequent updates on existing drafts.

Key Discussion Points

  • Document Status: Several documents are nearing completion and are ready for working group last call or publication. The chairs urged action to clear the document queue.

  • Draft Updates: Several drafts were presented with updates, including:

    • D-ALE for multi-homed traffic.
    • L2 Gateway Protocol.
    • EVPN with SRv6 support.
    • Extended optimized ingress replication.
    • EVPN First-Hop Security.
    • Unknown Mac mobility.
    • EVPN L2 Attributes and communities.
    • EVPN SRv6 Security
    • EVPN Route type one and three for bump traffic filtering.
    • EVPN inter-domain Option B.
    • SRv6 Anycast and Bypass SIDs
    • Upstream Designated Forwarder Selection.

  • EVPN First-Hop Security (Krishna): Discussion focused on the use of DHCP snooping and a new route type for handling multi-homing and host mobility. Concerns were raised about altering route reflection rules.

  • Unknown Mac Mobility (Hari): Presentation covered a procedure to improve Mac address scaling by limiting learning to the local network. Discussion included the need for support from all leaf nodes and the introduction of a second sequence counter for intra-network moves.

  • EVPN L2 Attributes and Communities (Hari): Clarifications were presented around route processing, mismatch behavior, signaling primary/backup PEs, and VLAN bundle service. Backward compatibility was a key concern.

  • SRv6 Anycast and Bypass SIDs (Shin): A new proposal to add A and B flags to SRv6 SIDs. The proposal addresses problems with local protection when using Anycast, but requires extensive review.
