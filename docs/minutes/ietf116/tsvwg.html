Session Date/Time: 30 Mar 2023 07:30

tsvwg

Summary

The second tsvwg meeting covered several topics, including detail security for SCTP, zero checksums in SCTP, SCTP authentication, SCTP over UDP, shared IP solutions, and media header extensions for wireless networks. The main discussion revolved around the direction of detail security for SCTP, with concerns raised about implementation challenges. The working group also discussed the adoption of the zero checksum draft. Several requests were made for feedback and reviews on the presented drafts.

Key Discussion Points

  • Detail Security for SCTP (Magnus):
    • Alternatives to detail over SCTP, focusing on well-known, maintained, and individually used security protocols.
    • Concerns raised by Michael regarding implementation complexity in the kernel.
    • Discussion on handling restarts and dynamic configuration with the proposed detail security solution.
    • Need for feedback from 3GPP and other stakeholders regarding the proposed changes to detail security.
  • Zero Checksums on SCTP (Michael):
    • Motivation: Saving CPU cycles when running SCTP over DTLS, where DTLS provides sufficient protection.
    • Clarification that the proposal is about CPU savings when the checksum doesn't provide additional value.
    • Adoption call: Working group agreed to adopt the draft, pending further review and addressing concerns on the mailing list.
  • SCTP Authentication (Michael):
    • Updating the authentication draft to address issues raised by Eric and incorporate socket API considerations.
    • Magnus highlighted the dependency of this draft on the decision of the direction of security for SCTP.
  • SCTP over UDP (Michael):
    • Integrating text submitted for Rc 6951 into a single document for SCTP with UDP.
    • Call for reviewers for this draft.
  • Shared IP Solutions (Lewis Chan):
    • Problem: How to enhance user experience on services running over shared IPv4 addresses (CGNAT).
    • Presentation of the stateless cgNAT solution: a novel approach for port forwarding without third party coordination.
    • Discussion deferred to the mailing list due to time constraints.
  • Media Header Extensions for Wireless Networks (John):
    • Problem: Poor media quality due to transient channel quality drops in wireless networks.
    • Proposal: Using a UDP option for providing metadata (priority and burst length) for packet scheduling.
    • Request for feedback and suggestions on the proposed solution.

Decisions and Action Items

  • Decision: The working group agreed to adopt the zero checksum draft (Michael), pending further review and addressing concerns on the mailing list.
  • Action Item: Magnus to engage with 3GPP and other stakeholders for feedback on the detail security proposal.
  • Action Item: Working group to discuss the direction of security for SCTP and conduct a consensus call on the mailing list.
  • Action Item: Solicit reviewers for the SCTP over UDP draft (Michael).
  • Action Item: Request feedback on Lewis Chan's cgNAT proposal on the mailing list.
  • Action Item: Working group members to review John's media header extensions draft and provide feedback.

Next Steps

  • Conduct a consensus call on the mailing list regarding the future direction of detail security for SCTP.
  • Gather feedback on the presented drafts from the working group and relevant external parties.
  • Address comments and issues raised in the discussions and incorporate them into the respective drafts.
  • Continue discussion on the mailing list for topics that could not be fully covered during the meeting.
