Session Date/Time: 30 Mar 2023 00:30

lake

Summary

This was a joint session between the LAKE and ACE working groups. The LAKE portion focused on the ad hoc draft, discussions around re-keying, and the Lake charter. The ACE portion covered updates on the pubs up profile, group notification, admin interface, and the D and now also TLS profile. The session included discussions on protocol design, security considerations, and future work items.

Key Discussion Points

  • ad hoc draft: Discussion of changes in versions 18 and 19, including the removal of the pad field, revision of the E syntax, and clarification of terminology. A new error code proposal related to credential identifiers was discussed.
  • Error Handling: Debate around the specification of error codes within ad hoc.
  • Exporter Usage: Discussion on how the exporter is used, with concerns about key handling, especially related to message three.
  • COSE Header Parameters: Discussion of new COSE header parameters for C simple web tokens and CCS, emphasizing the need for security considerations, particularly for verifying public keys.
  • Lake Traces: Agreement that the lake traces are ready for a working group last call.
  • Re-Keying: Discussion of a new P method for re-keying to address identity protection, tracking, and fingerprint printing.
  • Lake Charter: Review and discussion of the draft Lake charter, with emphasis on the scope of work and a rephrasing of the text about specifying how to use Ea fields.
  • ACE Pubs up Profile: Updates on the profile, including architectural changes, client group communication interface, and the distinction between application groups versus security groups. Also discussion regarding the use of scope and how they are used and implemented in the profile.
  • ACE Group Notification: Marco provided a quick update. Discussed the last call and the next steps.
  • ACE Admin Interface: Marco provided the summary of the updates in the past months. Discussions regarding the splitting of documents into two with new documents.
  • ACE OSCAR Profile: Deal spoke about the ACE OSCAR profile as well as its variants. There are now some discussion on the follow up activities for ACE OSCAR profile and their implementation.

Decisions and Action Items

  • ad hoc draft: Include the new error code proposal (P401) unless there are strong objections on the mailing list.
  • ad hoc draft: Include the existing proposal clarifying the proper ways to handle the session keys.
  • Lake Traces: Move forward with a working group last call on the lake traces draft, with a two-week last call period.
  • Lake Charter: Rephrase the text in the charter regarding specifying how to use Ea fields. The Chairs will work this into the data tracker.
  • ACE Sd Payload Profile: A call for adoption for this profile will be had, and one author is willing to lead the effort for completion.
  • ACE CP transport draft is under review by paul.
  • ACE heap draft is to be revied by the working group.
  • Confirm the new document from Ace Admin Francesco Peter prefers not to have the authors for these efforts to be shared.

Next Steps

  • LAKE: Start working group last call for the traces draft. Submit the charter to the datatracker.
  • ACE: Call for adoption for the Sd Payload Profile. Submit and address issues to C p2 transport draft. Review the heat Draft.
